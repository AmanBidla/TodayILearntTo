  9. How to sort a 1000 GB file with ram size is 4 GB only. Which algorithm or data structure we need to use to sort these files? Follow Up: External sort is Ok... but how can you make this solution more efficient... Follow Up 2: Ideal chunk size for external sort

    Use: a two pass approach
    First pass chunk sorting:

    1. Load chunks of 4 GB data in memory and do an in place sort (Merge Sort or Quick Sort).
    2. Write the chunk back to the same place on disk
    3. Repeat 250 times (until end of file or all data is consumed)

    Second Phases "merge": Now you have a choice to merge, what is going to be your "Bucket size" which will govern the N in a N-way-merge. (This gets real tricky, real fast) In this case we have 250 chunks of 4 GB each, One way to do this is,

    1. Load first 8.192 MB from each of 250 chunks (250*8.192 MB = 2GB) this is our input buffer and rest of the 4 GB is our output buffer
    2. Do a 250 ways merge on this 250 sorted bins / buckets
    3. When any of the 250 bins is empty load it with the next 8.192 MB from the associated chunks
    4. When Output buffer of 2 GB is full - write out the buffer to the external storage.

  21. 1 > Can you think how to implement stack and queue based on heap data structure ? It was wonderful when I was a reading the book on data structure and algorithms. As we all know stack in LIFO and when we use min heap , root element is always minimum and we can extract that in constant time. To implement stack which is LIFO, use an extra integer variable which will start with negative one or negative of current system time. Use above extra integer as priority when you insert into the min heap. You will get stack behavior. Comment on this blog if you need more details. But hope above sentence can give you enough idea. Queue as we all know is FIFO, element came at old time should be retrieved first. As we are using min heap, let us use another extra integer variable this time. But we will have positive initial value or positive system time. When we use that as priority to add elements to the min heap, element with recent time or max min value will be peeked every time.

    2 > Given two arrays , how will you find the paid which gives you maximum sum ? As I am writing this post under heap, you would have figured out already. Yes use two max-heaps one for each array and get the top most elements from both the heap and that will be your maximum sum.

    3 > How will you find median with heap ? We all know how median can be found. Given list of elements, sort them and return middle element or average of the middle two elements depending on even total or odd total number of elements. Think!!! We will have all elements before the median as less than median and after median will be greater than the median. So create two heaps. One is maxheap and another is min-heap using the input elements. If total number of elements are even then median is the average of maximum element from max heap and minimum element from the min heap. If total number of elements are odd then median is just the maximum element of the max heap.
